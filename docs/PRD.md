# PRD: 블로그 + 포트폴리오 통합 사이트

> Notion을 CMS로 활용한 개인 포트폴리오 및 블로그 웹 서비스

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 블로그 + 포트폴리오 통합 사이트 |
| 목적 | Notion API를 활용한 개인 포트폴리오 및 블로그 서비스 |
| 대상 사용자 | 개발자 본인 (콘텐츠 관리), 방문자 (콘텐츠 소비) |
| 작성일 | 2026-03-23 |

### 1.1 CMS로 Notion을 선택한 이유

- Notion API를 통해 별도의 관리자 UI 없이 콘텐츠 관리 가능
- 비개발자도 Notion 인터페이스에서 직관적으로 글 작성 및 수정 가능
- 별도 데이터베이스 서버 구축 불필요 — 운영 비용 최소화
- 풍부한 블록 타입 지원으로 다양한 콘텐츠 표현 가능

## 4. Notion 데이터베이스 구조

### 4.1 블로그 포스트 데이터베이스

| 필드명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| Title | title | 글 제목 | "Next.js로 블로그 만들기" |
| Category | select | 카테고리 | "개발", "일상", "독서" |
| Tags | multi_select | 태그 목록 | ["Next.js", "React", "TypeScript"] |
| Published | date | 발행일 | 2026-03-23 |
| Status | select | 발행 상태 | "발행됨" / "초안" |
| Content | page content | 본문 | Notion 페이지 블록 |

> **Status 필드 기준**: `발행됨` 상태인 글만 사이트에 노출

## 5. 화면 구성 (페이지 구조)

```
/                       홈 - 최근 글 목록 (6~9개)
/blog                   전체 글 목록
/blog/[slug]            글 상세 페이지
/blog/category/[name]   카테고리별 글 목록
/about                  내 소개 + 이력 타임라인
```

### 5.1 홈 (`/`)

- 최근 발행된 글 목록 (카드 형태)
- 간단한 자기소개 섹션
- 카테고리 빠른 이동 링크

### 5.2 글 목록 (`/blog`)

- 전체 발행된 글 목록
- 카테고리 필터 탭
- 검색 입력창
- 페이지네이션 또는 무한 스크롤

### 5.3 글 상세 (`/blog/[slug]`)

- 제목, 발행일, 카테고리, 태그 표시
- Notion 페이지 블록 렌더링
  - 텍스트, 제목, 이미지, 코드 블록, 구분선 등
- 이전/다음 글 네비게이션

### 5.4 카테고리별 목록 (`/blog/category/[name]`)

- 선택된 카테고리의 글 목록
- 카테고리명 헤더 표시

### 5.5 소개 페이지 (`/about`)

- 프로필 이미지 및 자기소개
- 기술 스택 배지
- 이력 타임라인 (학력, 경력, 프로젝트)
- 연락처 링크 (GitHub, 이메일 등)

## 7. 구현 단계

### 1단계: 환경 설정

1. Notion API 패키지 설치 (`@notionhq/client`)
2. Notion Integration 생성 및 API 키 발급
3. Notion 데이터베이스 생성 및 Integration 연결
4. `.env.local`에 환경 변수 설정

```env
NOTION_API_KEY=secret_xxxx
NOTION_DATABASE_ID=xxxx
```

### 2단계: Notion API 연동

1. Notion 클라이언트 초기화 유틸리티 작성
2. 글 목록 조회 함수 구현
3. 글 상세 조회 함수 구현
4. 블록 타입별 렌더링 컴포넌트 작성

### 3단계: 페이지 구현

1. 글 목록 페이지 (`/blog`)
2. 글 상세 페이지 (`/blog/[slug]`)
3. 홈 페이지 (`/`)

### 4단계: 스타일링 및 최적화

1. shadcn/ui 컴포넌트 적용
2. 반응형 레이아웃 구성
3. 이미지 최적화 (Next.js Image)
4. 메타데이터 및 SEO 기본 설정

### 5단계: 추가 기능 (MVP 이후)

1. 카테고리 필터링
2. 검색 기능
3. 소개 페이지 및 타임라인

## 9. 제약 사항

- Notion API 요청 속도 제한: 평균 3 req/s — ISR(Incremental Static Regeneration) 또는 캐싱 전략 필요
- Notion 무료 플랜 사용 시 API 통합 제한 확인 필요
- 이미지는 Notion에서 제공하는 S3 URL 사용 — URL 만료 이슈 고려 필요
